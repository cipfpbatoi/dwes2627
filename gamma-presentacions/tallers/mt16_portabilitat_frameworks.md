# Brief Gamma. MT16. Portabilitat entre frameworks

## Objectiu

Crear una micro-presentació opcional per a `R3S2B` que ajude a comparar responsabilitats comunes entre Laravel, Symfony, NestJS o un altre stack autoritzat.

## Missatge clau

Canviar de framework no hauria de canviar el problema. La portabilitat és entendre què fa cada capa encara que canvie el nom.

## Format recomanat

- 6 diapositives.
- Duració aproximada: `60 min`.
- Incloure una taula d'equivalències.
- Tancar amb una responsabilitat comparada.

## Estructura recomanada

1. Idea central: mateix problema, peces diferents.
2. Què es compara: rutes, controladors, models, serveis, validació, migracions i proves.
3. Què no es compara: quin framework és millor, sintaxi superficial, preferències i instal·lacions buides.
4. Taula d'equivalències: operació, peça equivalent, responsabilitat, dificultat i decisió.
5. Evidència del dia: taula de portabilitat, flux localitzat, decisió comparada i limitació.
6. Tancament: quina responsabilitat canvia de nom però no de sentit?

## Evidència mínima

- Taula de portabilitat.
- Flux localitzat o documentat.
- Decisió tècnica comparada.
- Limitació reconeguda.

## No incloure

- Tutorial de tres frameworks.
- Defensa d'un framework concret.
- Comparació sense projecte propi.
