# Brief Gamma. MT11. Provar i documentar una API

## Objectiu

Crear una micro-presentació de taller per a `R4S5` que ajude l'alumnat a alinear documentació, peticions reals i respostes de l'API.

## Missatge clau

Una API no està acabada si només funciona en el navegador. La documentació i el comportament real han de coincidir.

## Format recomanat

- 6 diapositives.
- Duració aproximada: `60 min`.
- Incloure exemples amb `curl`, Postman, Bruno o script curt.
- Tancar amb peticions reproduïbles.

## Estructura recomanada

1. Idea central: documentació executable.
2. Prova mínima: petició correcta, resposta, credencial, cas sense permís, dada incorrecta i instrucció.
3. Eines: `curl`, Postman, Bruno, test automatitzat, script o col·lecció.
4. Documentació útil: URL base, endpoints, headers, bodies, codis d'estat, errors i límits.
5. Evidència del dia: dues peticions reproduïbles, un error, README o contracte actualitzat i prova executada.
6. Tancament: quina petició pot copiar una altra persona per verificar la teua API?

## Evidència mínima

- Dues peticions reproduïbles.
- Un cas d'error.
- Documentació alineada amb la resposta real.
- Prova executada després del canvi.

## No incloure

- Documentació ornamental.
- Captures sense peticions repetibles.
- Estàndards avançats d'API no necessaris.
