# Brief Gamma. MT02. README executable i commit defensable

## Objectiu

Crear una micro-presentació de taller per a `R1S2` que ajude l'alumnat a deixar un repositori que es puga arrancar i entendre sense explicació oral.

## Missatge clau

El repositori ha de parlar abans de la defensa: un README executable i un historial de commits coherent mostren procés i autoria.

## Format recomanat

- 6 diapositives.
- Duració aproximada: `45 min`.
- Incloure exemples de README mínim i missatges de commit.
- Tancar amb una comprovació real d'arrencada.

## Estructura recomanada

1. Idea central: README i commits com a evidència.
2. README mínim: requisits, instal·lació, arrencada, URL, prova ràpida, estat i limitacions.
3. Commit defensable: objectiu clar, canvi coherent, estat raonable i frase explicable.
4. Errors habituals: README copiat, instruccions locals, commits gegants i missatges buits.
5. Evidència del dia: README provat, punt funcional verificat, commit concret i limitació anotada.
6. Tancament: quin commit explica millor què has aconseguit hui?

## Evidència mínima

- README amb arrencada real.
- Comanda o URL de comprovació.
- Commit amb missatge concret.
- Nota de limitacions si aplica.

## No incloure

- Guia extensa de Git.
- Convencions professionals complexes.
- Automatització de CI/CD.
