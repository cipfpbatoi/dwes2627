# Brief Gamma. Repte 5. Integració híbrida i tancament

## Objectiu

Crear una presentació de llançament del `Repte 5`, centrada en integrar una font o servei extern dins del producte per generar una funcionalitat híbrida amb valor real.

## Audiència

Alumnat de DWES que arriba de `R4` amb una API o servei documentat, provat i consumible.

## Missatge clau

`R5` no és cridar una API externa ni fer una automatització decorativa. És seleccionar una font útil, entendre'n el contracte, recuperar resposta real, transformar-la, integrar-la en el producte, provar errors i defensar el valor aportat.

## Estructura proposada

1. Què canvia en `R5`
   - De servei propi a integració híbrida.
   - De resposta externa a valor de producte.
2. Punt de partida
   - Producte persistent.
   - API o funcionalitat reutilitzable.
   - Documentació i proves de `R4`.
3. Microreptes del repte
   - `R5M1`: font externa i mapa d'integració.
   - `R5M2`: connexió real.
   - `R5M3`: mapping i transformació.
   - `R5M4`: funcionalitat híbrida integrada.
   - `R5M5`: prova, documentació i defensa.
4. Evidències mínimes
   - Comparativa de fonts, connector, mapping, demo completa, cas d'error i documentació.
5. Què no és suficient
   - JSON extern en cru, automatització ornamental, secrets exposats o defensa no verificable.
6. Connexió amb defensa final
   - El flux híbrid és una evidència de criteri, autoria i comprensió tècnica.

## To

Pràctic, de tancament de producte i verificació professional. Ha de sonar com una fase final real, no com una teoria genèrica d'APIs externes.

## Elements visuals suggerits

- Diagrama: producte propi -> font externa -> connector -> mapping -> funcionalitat final.
- Taula de microreptes amb pes i evidència.
- Exemple de comparativa font externa / resultat transformat.
- Flux de prova: cas correcte, error extern i defensa.

## No incloure

- Fer de `R5` una repetició de `R4`.
- Exigir integracions múltiples com a mínim obligatori.
- Convertir n8n, webhooks o una tecnologia concreta en requisit universal.
