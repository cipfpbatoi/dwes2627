# Brief Gamma. R5. Integració híbrida i defensa final

## Objectiu

Crear una presentació de llançament del `Repte 5` per explicar com integrar una font o servei extern dins del producte i convertir dades externes en una funcionalitat híbrida amb valor real, proves i defensa tècnica.

## Audiència

Alumnat que arriba de `R4` amb una API o servei documentat, provat i consumible.

## Missatge clau

`R5` no és cridar una API externa i mostrar-ne la resposta en cru. És seleccionar una font útil, entendre'n el contracte, connectar de manera controlada, transformar dades, integrar-les en el producte, provar errors i defensar el valor aportat.

## Estructura recomanada

1. **Portada**
   - `R5. Integració híbrida i defensa final`.
   - Subtítol: del servei propi al producte connectat.
2. **Punt de partida**
   - Producte persistent.
   - API pròpia o funcionalitat reutilitzable.
   - Documentació i proves de `R4`.
3. **Què canvia en R5**
   - Apareix una frontera externa.
   - Les dades ja no depenen només del nostre sistema.
   - Cal controlar disponibilitat, format, secrets i errors.
4. **Objectiu del repte**
   - Triar una font externa amb criteri.
   - Connectar amb resposta real.
   - Fer mapping i transformació.
   - Integrar el resultat dins del producte.
   - Provar i defensar el flux complet.
5. **Microreptes**
   - `R5M1`: font externa i mapa d'integració.
   - `R5M2`: connexió real.
   - `R5M3`: mapping i transformació.
   - `R5M4`: funcionalitat híbrida integrada.
   - `R5M5`: prova, documentació i defensa.
6. **Mapa d'integració**
   - Necessitat del producte.
   - Font externa.
   - Dades disponibles.
   - Transformació.
   - Ús dins del producte.
   - Riscos i límits.
7. **Evidències principals**
   - Comparativa de fonts.
   - Connector o petició real.
   - Exemple de resposta externa.
   - Taula de mapping.
   - Funcionalitat integrada.
   - Cas d'error.
   - Documentació i defensa.
8. **Què no és suficient**
   - Mostrar `JSON` extern sense transformar.
   - Fer una automatització ornamental.
   - Exposar claus o secrets.
   - Dependre d'una resposta externa sense pla d'error.
   - Inventar la defensa després sense evidències.
9. **Defensa final**
   - Quin valor aporta la integració?
   - Què entra del sistema extern?
   - Què es transforma?
   - Què es mostra o activa en el producte?
   - Com es prova el cas correcte i el cas d'error?
10. **Checklist final**
    - La font externa està justificada.
    - La connexió és real i reproduïble.
    - El mapping està documentat.
    - La integració apareix dins del producte.
    - Hi ha tractament d'error.
    - La defensa explica contracte, valor, límits i autoria.

## Format recomanat

- 10-12 diapositives.
- Diagrama: producte propi -> font externa -> connector -> mapping -> funcionalitat final.
- Taula de microreptes amb evidència principal.
- Exemple de dada externa i resultat transformat.
- Flux de prova: cas correcte, error extern i defensa.
- Tancar amb checklist final del repte.

## No incloure

- Fer de `R5` una repetició de `R4`.
- Exigir integracions múltiples com a mínim obligatori.
- Convertir n8n, webhooks o una tecnologia concreta en requisit universal.
- Teoria genèrica d'APIs externes sense connexió amb el producte.
