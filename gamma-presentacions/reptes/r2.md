# Brief Gamma. R2. Processament, estat i autenticació

## Objectiu

Crear o actualitzar una presentació de repte per a `R2` que introduïsca el pas des d'una base executable cap a una primera funcionalitat real de backend: entrada de dades, validació, reintent, guardat funcional, regles del projecte, estat i autenticació.

## Contingut que ha d'incloure

- Context professional: un backend no només mostra pàgines, també rep dades, valida, decideix, recorda estat i protegeix operacions.
- Connexió amb `R1`: ja hi ha base executable; ara cal convertir-la en flux funcional.
- Objectiu del repte: construir una primera funcionalitat del producte amb recorregut de servidor complet.
- Seqüència de microreptes:
  - `R2M1`: entrada variada i validació bàsica de servidor.
  - `R2M2`: processament, reintent i guardat funcional.
  - `R2M3`: lògica del flux i regles del projecte.
  - `R2M4`: estat, sessió i/o cookies.
  - `R2M5`: autenticació i funcionalitat protegida.
  - `R2M6`: proves, depuració i documentació mínima.
  - `R2M7`: primera peça testable amb POO, Composer i prova unitària.
- Evidències principals del repte.
- Criteris d'avaluació treballats: `RA2`, `RA3` i `RA4`.
- Errors habituals:
  - començar per login sense tindre flux de dades;
  - validar només en client;
  - guardar dades incorrectes;
  - usar sessió com si fora persistència formal;
  - protegir una pàgina decorativa en lloc d'una operació real.
- Connexió amb `R3`: el flux funcional es reconstruirà amb millor arquitectura i persistència real.

## Criteris de disseny

- Presentació de 10-12 diapositives.
- Començar amb un esquema de flux: formulari -> servidor -> validació -> resposta -> reintent -> guardat -> regla -> estat -> autenticació.
- Usar exemples de producte, no exemples genèrics sense domini.
- Incloure una diapositiva final de checklist del repte.
- No convertir la presentació en una guia de codi completa.

## Límit

No avançar encara arquitectura MVC, ORM, migracions, APIs ni integracions. Això pertany als reptes posteriors.
