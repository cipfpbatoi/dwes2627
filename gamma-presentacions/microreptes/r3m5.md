# Brief Gamma. R3M5. Autenticació, autorització i middleware

## RA avaluat

- **RA avaluat**: `RA5`
- **CA avaluats**: `RA5.f`, `RA5.g`, `RA5.h`
- **RA de context**: `RA6.f`, `RA8.g`

Este microrepte genera nota només per al RA avaluat. Els RA de context poden aparéixer en el producte, però no es ponderen en esta rúbrica.


## Objectiu

Presentar el microrepte `R3M5`: protegir una acció significativa de `R3` amb autenticació del framework, autorització mínima i middleware, guard, voter, policy o mecanisme equivalent.


## Contingut

- Codi: `R3M5`; sessió: `R3S5`.
- Diferència entre autenticació i autorització.
- Per què ocultar un botó no protegeix una ruta.
- Equivalències:
  - `Laravel`: middleware, guard, policy;
  - `Symfony`: security, access control, voter;
  - `NestJS`: guard, middleware, decorator, pipe.
- Què ha de construir:
  - login o mecanisme equivalent;
  - logout o invalidació;
  - usuari demo;
  - acció protegida;
  - regla d'autorització mínima;
  - cas autoritzat;
  - cas denegat;
  - instruccions de prova.
- Errors habituals: protecció només visual, rols sense ús, no provar cas denegat, secrets reals versionats.

## Límit

No demanar OAuth, 2FA, sistema complet de rols, JWT avançat ni API completa.
