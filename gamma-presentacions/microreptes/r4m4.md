# Brief Gamma. R4M4. Consum real del servei

## Objectiu

Presentar el microrepte `R4M4`: demostrar que l'API es pot consumir des d'una eina, script, client o peça equivalent.

## Punts clau

- Provar el servidor no és el mateix que consumir el servei.
- El consumidor ha de fer una petició real.
- Cal tractar almenys un cas d'error.
- Les instruccions han de permetre repetir la prova.

## Evidències

- Consumidor mínim o col·lecció configurada.
- Cas correcte.
- Cas d'error.
- Instruccions de reproducció.
- Contrast amb el contracte.

Què necessitaria una altra persona per consumir la teua API sense preguntar-te?
