# Brief Gamma. R2M7. Proves, depuració i checkpoint tècnic

## RA avaluat

- **RA avaluat**: `RA4`
- **CA avaluats**: `RA4.f`
- **RA de context**: cap

Este microrepte genera nota només per al RA avaluat. Els RA de context poden aparéixer en el producte, però no es ponderen en esta rúbrica.


## Objectiu

Crear o actualitzar una presentació de microrepte per a `R2M7`, centrada en provar el flux complet de `R2`, registrar resultats reals, depurar incidències i deixar una demo reproduïble.


## Contingut que ha d'incloure

- Codi del microrepte: `R2M7`.
- Sessió associada: `R2S7`.
- Per què provar no és només fer una demo del camí feliç.
- Bateria mínima:
  - cas vàlid;
  - cas invàlid;
  - reintent;
  - regla amb dos resultats;
  - estat recuperat i invalidat;
  - accés no autenticat denegat;
  - accés autenticat permés;
  - logout o invalidació.
- Com registrar entrada, passos, resultat esperat, resultat real i incidència.
- Diferència entre prova manual, prova automàtica lleugera de flux i prova unitària.
- Mínim automàtic: una comanda `curl`, script `.sh`, script PHP senzill o col·lecció de peticions documentada.
- Exemples:
  - ruta protegida sense sessió retorna denegació;
  - login correcte permet una operació protegida;
  - formulari invàlid mostra error.
- Errors habituals: proves no executades, README desactualitzat, corregir sense reproduir, no provar denegacions.
- Checklist final.

## Límit

No demanar proves unitàries, `PHPUnit`, mocks ni suite formal com a mínim. Sí que cal una prova automàtica lleugera de flux.
