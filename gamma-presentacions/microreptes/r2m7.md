# Brief Gamma. R2M7. Primera peça testable amb POO i Composer

## Objectiu

Crear o actualitzar una presentació de microrepte per a `R2M7`, centrada en crear una primera classe simple carregada amb Composer i verificar-la amb una prova unitària mínima.

## Referències existents

- `docs/02_reptes/repte_02.md`
- `dwes-restructuracio-modul/docs/01_programacio_modul/programacio_aula_r2s7_primera_peca_testable_poo_composer.md`
- `dwes-microreptes-autocorreccio/microreptes/r2-s07-primera-peca-testable-poo-composer/`

## Contingut que ha d'incloure

- Codi del microrepte: `R2M7`.
- Sessió associada: `R2S7`.
- Diferència entre prova unitària i prova del flux web.
- Per què Composer ajuda a carregar classes pròpies.
- Per què la classe ha de poder provar-se sense navegador.
- Què ha de fer l'alumnat:
  - triar una regla, comprovació o càlcul del flux ja provat;
  - crear o ajustar `composer.json`;
  - configurar autoload;
  - crear una classe simple en `src/` o equivalent;
  - escriure una prova unitària mínima amb `PHPUnit`, `Pest` o script equivalent;
  - executar la prova i documentar el resultat;
  - repetir un cas clau del flux web;
  - indicar què queda per a `R3`.
- Errors habituals: classe que només conté HTML, classe dependent de `$_POST` o `$_SESSION`, Composer configurat però no usat, prova que no s'executa, intentar fer MVC complet.
- Checklist final.

## Límit

No demanar MVC complet, ORM, migració a framework, POO extensa, API externa ni base de dades com a focus central.
