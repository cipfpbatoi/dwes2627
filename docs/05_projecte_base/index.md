# Projecte base

## Idea general

Durant el curs no faràs un projecte diferent en cada repte.

Treballaràs sobre un mateix producte backend que ha d’arribar viu de `R1` a `R5`.

## Com ha d’evolucionar

El recorregut esperat és este:

- `R1`: arranque del producte i primer flux funcional
- `R2`: continuació sobre una base comuna en `PHP`
- `R3`: entrada de framework sense canviar de projecte
- `R4`: publicació i consum d’API
- `R5`: integració, manteniment i tancament

A partir del framework, el mateix projecte pot continuar en:

- `Laravel`
- `Symfony`
- `NestJS`

`FastAPI` només pot aparéixer com a via avançada o excepcional, no com a itinerari base general.

## Com triar bé el projecte

Convé que el teu projecte permeta:

- treballar dades i lògica de servidor
- introduir autenticació i funcionalitat protegida
- passar després a persistència
- exposar una API útil
- arribar a una integració o automatització final

No convé triar un cas massa ornamental o massa estret, perquè després el curs es queda sense recorregut.

## Mateix encàrrec funcional

Quan el curs parla de continuar el mateix projecte, vol dir que el problema funcional continua sent el mateix encara que canvies d’estructura o de framework.

El que evoluciona és la solució tècnica, no l’encàrrec.

## Si treballes amb altres companys

El model preferent és compartir projecte i encàrrec funcional, però mantindre:

- repositori individual
- evidències individuals
- defensa individual

La cooperació servix per contrastar decisions i comparar tecnologies, no per dissoldre l’autoria.
