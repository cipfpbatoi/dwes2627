# Rúbrica R4M3. Protecció de l'API

| Dimensió | Pes | Què es comprova |
|---|---:|---|
| Resolució funcional | `35%` | Control d'accés real i observable. |
| Verificació | `25%` | Cas autoritzat, sense credencial i credencial incorrecta. |
| Traçabilitat | `10%` | Registre del mecanisme i de les proves. |
| Documentació | `10%` | S'explica com enviar credencial i què passa si falla. |
| Ús d'IA | `5%` | Si hi ha IA, es justifica i es verifica. |
| Qualitat tècnica | `15%` | El control està en el flux correcte. |

## Regles dures

- Sense control d'accés real, màxim recomanat `5`.
- Sense cas denegat provat, màxim recomanat `7`.
- Si el cas denegat retorna dades sensibles, cal revisió docent.
