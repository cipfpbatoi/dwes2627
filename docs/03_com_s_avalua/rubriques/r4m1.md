# Rúbrica R4M1. Disseny del contracte del servei

## RA avaluat

- **RA avaluat**: `RA7`
- **CA avaluats**: `RA7.a`, `RA7.b`, `RA7.c`, `RA7.d`
- **RA de context**: cap

Este microrepte genera nota només per al RA avaluat. Els RA de context poden aparéixer en el producte, però no es ponderen en esta rúbrica.


| Dimensió | Pes | Què es comprova |
|---|---:|---|
| Resolució funcional | `30%` | Contracte amb recurs real, endpoints, request/response i errors. |
| Verificació | `20%` | El contracte encaixa amb el producte i amb casos d'ús possibles. |
| Traçabilitat | `10%` | Issue, registre o commits de decisió del servei. |
| Documentació | `20%` | Documentació clara de formats, errors i valor del servei. |
| Ús d'IA | `5%` | Si hi ha IA, està contrastada i registrada. |
| Qualitat tècnica | `15%` | Evita endpoints arbitraris i no desborda l'abast de `R4`. |

## Regles dures

- Sense contracte documentat, màxim recomanat `5`.
- Si el recurs no ve del projecte real, màxim recomanat `6`.
- Sense errors o codis d'estat, màxim recomanat `7`.
