# R3M2. Persistència mínima amb migrations i seeders

## Visor del microrepte

| Camp | Valor |
|---|---|
| Repte | `R3` |
| Sessió | `R3S2` |
| Microrepte | `R3M2` |
| Pes dins del repte | `20%` |
| Autocorrecció | `r3-s02-persistencia-minima-migrations-seeders` |

## Criteris i pesos

| Dimensió | Pes | Què es comprovarà |
|---|---:|---|
| Resolució funcional | `30%` | BBDD real amb migrations, seeders i lectura de dades. |
| Verificació | `20%` | Reset o reconstrucció de BBDD, càrrega i lectura comprovades. |
| Traçabilitat | `10%` | Commits o registre de model, migracions, seeders i prova. |
| Documentació | `15%` | Com crear BBDD, carregar dades i quin flux alimenten. |
| Ús d'IA | `5%` | Registre i validació de propostes de camps o migracions. |
| Qualitat tècnica | `20%` | Esquema mínim, coherent i no manual. |

## Evidències requerides

- Connexió a BBDD.
- Migrations.
- Seeders, fixtures o equivalent.
- Dades inicials reproduïbles.
- Lectura real de conjunt de dades.
- Ús o preparació del conjunt en un flux.

## Fora d'abast

No es demanen encara dos fluxos complets ni API.
