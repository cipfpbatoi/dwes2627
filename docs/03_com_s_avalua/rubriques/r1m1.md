# R1M1. Guia d'avaluació

## Visor del microrepte

| Camp | Valor |
|---|---|
| Repte | `R1. Kickoff funcional del backend` |
| Sessió | `R1-S01` |
| Microrepte | `R1M1` |
| Títol | Model client/servidor i elecció guiada de stack |
| Pes dins del repte | `25%` |
| Autocorrecció associada | `r1-s01-model-client-servidor-stack` |
| Producte esperat | Fitxa breu d'exploració tècnica inicial, explicació client/servidor i decisió tècnica orientativa. |
| Finalitat | Demostrar que entens el paper del backend i que pots prendre una primera decisió tècnica coherent amb el projecte i amb el curs. |

## Criteris d'avaluació

| Dimensió | Pes | Què es comprovarà |
|---|---:|---|
| Resolució funcional | `35%` | La fitxa resol el model client/servidor del producte i formula una decisió tècnica orientativa coherent. |
| Verificació | `20%` | Hi ha evidència revisable de la fitxa, fonts o contrast utilitzat, i punts que pots defensar oralment. |
| Traçabilitat | `10%` | Els documents i decisions del microrepte són localitzables al repositori i no queden només en converses externes. |
| Documentació | `15%` | El `README` o la documentació enllaça la fitxa, resumeix la decisió i deixa clars els dubtes oberts. |
| Ús d'IA | `5%` | Si uses IA, el resultat mostra contrast i comprensió pròpia; les captures o respostes d'IA no substituïxen la fitxa. |
| Qualitat tècnica | `15%` | La comparació de tecnologies és acotada, realista i alineada amb el recorregut del curs. |

## Evidències requerides

| Evidència | Què ha d'incloure |
|---|---|
| Fitxa breu d'exploració tècnica inicial | Model client/servidor aplicat al teu producte, opcions valorades i decisió inicial. |
| Explicació client vs servidor | Diferència clara entre què s'executa al navegador i què s'executa al servidor. |
| Primera decisió tècnica orientativa | Tecnologia o stack inicial, motiu de la tria i encaix amb el projecte. |
| Alternativa descartada | Una opció no triada amb una raó curta i comprensible. |
| Dubte o límit tècnic real | Un punt que encara no tens resolt i que caldrà comprovar més avant. |
| Referència a fonts o IA, si n'hi ha | Fonts consultades o ús d'IA explicat de manera breu i verificable. |

## Senyals esperats

| Senyal | Què mostra |
|---|---|
| Diferencies navegador i servidor | Entens el model d'execució bàsic del backend. |
| Uses exemples del teu producte | La fitxa no és genèrica ni copiada d'una explicació externa. |
| Identifiques navegador, servidor web, runtime i backend | Pots situar cada peça dins del flux. |
| Compares opcions de manera acotada | No fas un informe llarg, sinó una decisió útil per començar. |
| Tanques opció preferent, alternativa descartada i dubte pendent | El microrepte deixa una decisió real i revisable. |

## Fora d'abast

| No es demana encara | Per què |
|---|---|
| Servidor executable | Es treballa en `R1M2`. |
| Punt d'entrada funcional | Es treballa en `R1M2`. |
| Framework complet | Encara estem en decisió inicial, no en implantació completa. |
| Formularis, validació, sessions, login o persistència | Pertanyen a reptes posteriors. |
| Captures d'IA com a lliurable principal | La IA pot ajudar, però la comprensió ha de ser teua. |

## Regles dures de revisió

| Situació | Conseqüència orientativa |
|---|---|
| No hi ha fitxa d'exploració tècnica ni document equivalent | La puntuació màxima recomanada és `5`. |
| No diferencies client i servidor aplicat al producte propi | La puntuació màxima recomanada és `6`. |
| Només aportes captures o text d'IA sense elaboració pròpia | Requereix revisió docent. |
| No hi ha `README` o documentació localitzable | S'aplicarà penalització de documentació i traçabilitat. |
