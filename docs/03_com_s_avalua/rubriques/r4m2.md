# Rúbrica R4M2. Primer endpoint funcional

| Dimensió | Pes | Què es comprova |
|---|---:|---|
| Resolució funcional | `35%` | Endpoint real amb dades o acció del projecte. |
| Verificació | `20%` | Cas correcte provat amb instruccions reproduïbles. |
| Traçabilitat | `10%` | Commits o registre del pas de contracte a implementació. |
| Documentació | `15%` | S'indica com executar la petició i què retorna. |
| Ús d'IA | `5%` | Si hi ha IA, el codi està verificat. |
| Qualitat tècnica | `15%` | Resposta estructurada i coherent amb el contracte. |

## Regles dures

- Sense endpoint funcional, màxim recomanat `5`.
- Si la resposta és estàtica i desconnectada, màxim recomanat `6`.
- Sense prova reproduïble, màxim recomanat `7`.
