# Rúbrica R5M5. Prova, documentació i defensa

| Dimensió | Pes | Què es comprova |
|---|---:|---|
| Resolució funcional | `25%` | Integració final tancada amb cas positiu i cas d'error. |
| Verificació | `25%` | Proves o verificacions reproduïbles del flux complet. |
| Traçabilitat | `10%` | Registre d'ajustos, incidències, commits i IA si correspon. |
| Documentació | `20%` | Font, configuració, mapping, execució, límits i mapa final. |
| Ús d'IA | `5%` | L'AI log reflectix ús real i validació, no relat fictici. |
| Qualitat tècnica | `15%` | El tancament no obri peces grans sense provar ni documentar. |

## Regles dures

- Sense prova positiva, màxim recomanat `5`.
- Sense prova d'error, màxim recomanat `7`.
- Sense mapa final o documentació, màxim recomanat `7`.
