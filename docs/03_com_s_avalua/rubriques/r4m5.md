# Rúbrica R4M5. Documentació i prova del servei

| Dimensió | Pes | Què es comprova |
|---|---:|---|
| Resolució funcional | `25%` | Servei tancat amb endpoints, autenticació, proves i consumidor identificables. |
| Verificació | `25%` | Proves amb cas correcte, error i contrast documentació-comportament. |
| Traçabilitat | `10%` | Incidències, correccions, commits o registre de tancament. |
| Documentació | `20%` | Documentació executable per una altra persona. |
| Ús d'IA | `5%` | Si hi ha IA, ajuda verificada i registrada. |
| Qualitat tècnica | `15%` | Contracte, implementació, protecció i consumidor són coherents. |

## Regles dures

- Sense documentació executable, màxim recomanat `6`.
- Sense proves reproduïbles, màxim recomanat `7`.
- Sense cas d'error, màxim recomanat `8`.
