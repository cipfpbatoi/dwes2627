# R2M6. Mini API d'autenticació per a client

## RA avaluat

- **RA avaluat**: `RA4`
- **CA avaluats**: `RA4.d`, `RA4.e`
- **RA de context**: `pont DWEC`

Este microrepte genera nota només per al RA avaluat. Els RA de context poden aparéixer en el producte, però no es ponderen en esta rúbrica.


## Què es valora

Este microrepte valora que l'autenticació creada en `R2M5` es puga consumir des d'un client extern mitjançant una mini API amb `JSON`, codis HTTP coherents i una ruta protegida.

No es valora encara tindre una API REST completa, OpenAPI, OAuth, JWT professional, refresh tokens ni arquitectura de `R4`.

## Evidències mínimes

- Endpoint `POST /api/login` o equivalent.
- Resposta `JSON`.
- Codi `200` en cas correcte i `401` en cas incorrecte.
- Token simple o mecanisme equivalent documentat.
- Endpoint protegit `GET /api/me`, `GET /api/protected` o equivalent.
- Comprovació real del token o mecanisme triat.
- Prova externa amb `curl`, Postman/Insomnia o `fetch`.
- Contracte documentat perquè DWEC el puga consumir.
- Usuari demo i limitacions conegudes, incloent CORS si afecta el consum.

## Rúbrica

| Dimensió | Pes | Què ha d'estar clar |
|---|---:|---|
| Login API i ruta protegida | 35% | Hi ha login API, token o mecanisme documentat, i ruta protegida que comprova realment l'accés. |
| Codis HTTP, JSON i prova externa | 20% | Les respostes són `JSON`, el cas correcte retorna `200`, el cas incorrecte o no autenticat retorna `401` i s'ha provat des d'una eina externa. |
| Traçabilitat | 10% | Hi ha commits o registre equivalent del pas de login web a contracte consumible. |
| Contracte per a DWEC | 20% | El contracte indica URL base, mètodes, headers, body, exemples `200`/`401`, usuari demo i limitacions conegudes. |
| Ús d'IA | 5% | Si s'ha usat IA, queda registrat què s'ha acceptat i com s'ha verificat. |
| Seguretat bàsica | 10% | No exposa contrasenyes ni hashes, i el token es comprova realment en la ruta protegida. |

## Regles dures

- Sense resposta `JSON`, la puntuació màxima recomanada és 7.
- Sense ruta protegida, la puntuació màxima recomanada és 7.
- Si retorna sempre `200`, la puntuació màxima recomanada és 8.
- Si exposa contrasenyes o hashes, cal corregir-ho abans de qualificar.
- Si el token es genera però no es comprova, la puntuació màxima recomanada és 7.
- Sense contracte mínim documentat per a DWEC, la puntuació màxima recomanada és 8.
- No exigisques API completa, OpenAPI, JWT professional, OAuth ni refresh tokens.
