# R2M7. Proves, depuració i checkpoint tècnic

## RA avaluat

- **RA avaluat**: `RA4`
- **CA avaluats**: `RA4.f`
- **RA de context**: cap

Este microrepte genera nota només per al RA avaluat.

## Què es valora

Es valora que el flux de `R2` deixe de dependre d'una demo oral i tinga proves mínimes reproduïbles, incidències registrades i documentació de reproducció.

No es demana encara una suite completa ni testing professional.

## Evidències mínimes

- Checklist o taula de proves amb entrada, passos, resultat esperat i resultat real.
- Cas vàlid executat.
- Cas invàlid per validació executat.
- Reintent amb dades conservades comprovat.
- Regla del projecte amb dos resultats comprovada.
- Estat recuperat i invalidat comprovat.
- Accés no autenticat denegat.
- Accés autenticat permés.
- Logout o invalidació i nou intent denegat.
- Almenys una prova automàtica lleugera amb `curl`, script, col·lecció o equivalent.
- Instruccions per executar la prova.
- Incidències detectades o confirmació explícita que no n'hi ha.
- README o guia de reproducció actualitzada.

## Rúbrica

| Dimensió | Pes | Què ha d'estar clar |
|---|---:|---|
| Resolució funcional | `25%` | El flux complet de `R2` és executable i revisat. |
| Verificació | `30%` | Hi ha bateria de proves i almenys una prova automàtica lleugera. |
| Traçabilitat | `10%` | Hi ha incidències, correccions o registre de checkpoint. |
| Documentació | `15%` | El README permet repetir les proves principals. |
| Ús d'IA | `5%` | Si s'ha usat IA, s'ha verificat el resultat de les proves. |
| Qualitat tècnica | `15%` | Les proves són simples, útils i connectades al flux real. |

## Regles dures

- Sense prova de cas denegat, màxim recomanat `8`.
- Sense cap prova reproduïble, màxim recomanat `6`.
- Sense instruccions d'execució, màxim recomanat `8`.
- Si el checkpoint trenca un flux anterior, cal corregir abans de tancar `R2`.
