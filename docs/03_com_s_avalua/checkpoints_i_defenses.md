# Checkpoints i defenses

## Què és un checkpoint

Un checkpoint és una revisió curta del teu estat de treball en un moment concret del repte.

En un checkpoint hauràs de poder mostrar:

- què tens fet
- què funciona de veritat
- què queda pendent
- quines decisions has pres
- quin serà el pas següent

## Què és una defensa tècnica

Una defensa tècnica és una explicació breu del teu propi treball.

En una defensa s’espera que pugues:

- mostrar el sistema funcionant
- explicar el flux funcional principal
- justificar decisions importants
- parlar d’errors i correccions
- explicar l’ús de la IA, si n’has fet
- respondre preguntes o fer un microcanvi

## Què has de tindre a mà

- el repositori actualitzat
- el `README` amb com reproduir el projecte
- una prova curta del cas correcte
- una prova curta d’una fallada rellevant
- una explicació clara d’on està la lògica principal

## Si cooperes

Pots arribar a un checkpoint amb contrast tècnic compartit o amb el mateix encàrrec funcional que altres companys.

Però cada persona ha de poder:

- obrir el seu repositori
- mostrar les seues proves
- explicar el seu propi codi
- defensar les seues decisions

## Regla curta

El checkpoint pot ser compartit en context. La defensa, l’evidència i la responsabilitat continuen sent individuals.
