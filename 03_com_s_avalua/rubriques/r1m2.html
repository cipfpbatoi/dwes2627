    R1M2. Guia d'avaluació¶

    RA avaluat¶

    • RA avaluat: RA1
    • CA avaluats: RA1b, RA1c, RA1d, RA1e, RA1f, RA1g
    • RA de context: cap

    Este microrepte genera nota només per al RA avaluat. Els RA de context poden aparéixer en el producte, però no es ponderen en esta rúbrica.

    Visor del microrepte¶

    Camp Valor
    Repte R1. Kickoff funcional del backend
    Sessió R1-S02
    Microrepte R1M2
    Títol Entorn executable, punt d'entrada i tancament de R1
    Pes dins del repte 75%
    Autocorrecció associada r1-s02-entorn-executable
    Producte esperat Repositori executable, primer punt d'entrada funcional, README, documentació i checkpoint de pas a R2.
    Finalitat Demostrar que pots convertir la decisió inicial en una base backend executable, verificable, documentada i defensable.

    Criteris d'avaluació¶

    Dimensió Pes Què es comprovarà
    Resolució funcional 35% L'entorn està definit i arranca amb Docker o equivalent, i hi ha una ruta, vista, endpoint, landing mínima o healthcheck que respon des del backend.
    Verificació 20% Inclous prova d'arrencada, URL, captura o log, i comprovació reproduïble del servei web, BBDD/phpMyAdmin quan corresponga i punt d'entrada funcional.
    Traçabilitat 10% Hi ha issue mare, microtasques, commits o registre de decisions vinculats a l'entorn, al punt d'entrada, la documentació i el tancament de R1.
    Documentació 15% El README és executable, indica requisits, arrencada, parada, ports, URL i comprovacions, i la documentació del repte és localitzable dins del repositori.
    Ús d'IA 5% Si uses IA per configurar, depurar o documentar, registres les consultes rellevants i verifiques que el resultat correspon al sistema real.
    Qualitat tècnica 15% La configuració i el punt d'entrada són simples, adaptats al projecte i explicables; no són una còpia opaca impossible de defensar.

    Evidències requerides¶

    Evidència Què ha d'incloure
    Configuració d'entorn docker-compose.yml, compose.yaml, Dockerfile o equivalent amb PHP, servidor web, BBDD i phpMyAdmin quan corresponga.
    Error documentat, si l'entorn no arranca Log, hipòtesi de causa i pla de correcció concret.
    README executable Requisits, arrencada, parada, serveis, ports, URL i resultat esperat.
    Punt d'entrada funcional Ruta, vista, endpoint, landing mínima o healthcheck servit pel backend.
    Evidència de comprovació Demo, captura, log o URL que permeta revisar l'entorn i el punt d'entrada.
    Documentació localitzable Fitxa de R1M1, decisió tècnica, incidències, verificacions i pendent de pas a R2.
    Justificació tècnica Decisió curta, ADR inicial o explicació equivalent.
    Traçabilitat Commit, issue, microtasques o registre associat a l'entorn i al punt d'entrada.
    Explicació del flux Petició, execució en servidor i resposta.
    AI log, si hi ha ús rellevant d'IA Consultes importants i verificació feta després.

    Senyals esperats¶

    Senyal Què mostra
    docker compose up o equivalent arranca els serveis L'entorn és reproduïble.
    El projecte està adaptat Nom, ports, estructura o resposta inicial no són una còpia plana.
    El README permet repetir l'arrencada Una altra persona pot provar el projecte sense explicacions orals.
    Hi ha una URL, ruta o endpoint que respon El backend ja genera una resposta visible.
    La resposta està connectada amb el producte No és només decoració ni una maqueta desconnectada.
    Identifiques què fa cada servei Pots explicar servidor web, PHP, BBDD i phpMyAdmin quan corresponga.
    La documentació està dins del repositori Les decisions i evidències són revisables.
    El tancament explica el pas cap a R2 El kickoff deixa una base clara per a formularis i validació.

    Fora d'abast¶

    No es demana encara Per què
    Formularis i validació de dades Comencen en R2M1.
    Sessions, login, rols o operacions protegides Pertanyen a microreptes posteriors.
    Persistència funcional rica Es treballarà quan el flux de dades ja estiga introduït.
    Framework complet obligatori El focus és una base executable i explicable, no una migració completa a framework.
    Funcionalitat de domini complexa Només cal un punt d'entrada simple i coherent amb el producte.

    Regles dures de revisió¶

    Situació Conseqüència orientativa
    No hi ha configuració d'entorn executable ni error documentat La puntuació màxima recomanada és 5.
    No hi ha punt d'entrada funcional localitzable La puntuació màxima recomanada és 6.
    No hi ha evidència de prova, captura, log o URL reproduïble La puntuació màxima recomanada és 6.
    No hi ha README executable o comprovable La puntuació màxima recomanada és 6.
    No hi ha documentació localitzable dins del repositori La puntuació màxima recomanada és 7.
    És només una pàgina estàtica sense relació amb el backend Requereix revisió docent.
    El projecte és una còpia plana no adaptada ni explicable Requereix revisió docent.

    Connexió amb la via 9→10¶

    El healthcheck pot servir com a punt d'entrada mínim per demostrar que l'entorn arranca i que el backend respon.

    Però, per optar a la franja 9→10 del Repte 1, no és suficient. La millora esperada és una primera experiència visible del producte:

    • landing page pròpia del projecte
    • pàgina Sobre nosaltres o Sobre el projecte
    • navegació mínima entre pàgines
    • contingut adaptat al producte
    • prova funcional i rutes documentades en el README

    Esta ampliació ha d'estar servida pel backend i ha de poder-se explicar en defensa breu.
